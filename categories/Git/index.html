Github 메인페이지 README.md 배지(badge)로 꾸미기

깃허브 메인페이지 README파일 노출 시키기

  • 위의 사진의 Hi there... 가 적힌 문구는 제 깃허브 아이디(InhwanCho)와 동일한 이름의 repo(저장소) InhwanCho
    README.md 파일입니다.(더블 클릭된 부분)
  1. 오른쪽 상단에 +를 눌러서 New Repository를 클릭해서 아이디(Owner)와 동일한 이름의 repo를 생성합니다.
  2. 아이디와 동일한 이름의 repo에 들어가서 README.md 파일을 생성합니다.
  3. 마크다운 문법을 사용하여 메인 페이지를 꾸며줍니다.

배지(badge) 사용 방법

  • 일단 위의 사진의 README.md 파일의 데이터입니다.
  • 맨위는 ###의 H3문법, 그 아래는 html 문법이 적용되었습니다.
  • 아이콘 및 컬러에 대한 정보는 <simpleicons> 를 들어가셔서 참고하시면 됩니다.
  • 예를들어 스텍스에 파이썬을 입력하고 싶다면 아래 대괄호 [] 부분만 변경하여 사용하시면 됩니다.

Github info

HEAD란?

  • 깃허브를 사용하다보면 HEAD라는게 자주 보입니다.
  • HEAD란 해당 branch의 마지막 commit을 의미합니다.

깃허브에 올리지 말아야 할 파일이 있을 경우

  • 깃허브를 사용하다보면 개인적으로 작업 또는 불필요한 파일들이 깃허브 폴더에 들어있을 경우가 있습니다.
  • 이를 제어하려면 .ignore파일이 필요합니다.
  • 주의할 점은 이미 stage/repo에 올라간 파일이 .gitignore파일에 있으면 오류가 난다.

Github commands

Github 연결

  • 맥 OS에는 기본적으로 git이 설치되어있습니다.

Github이랑 컴퓨터랑 연동(처음 등록할 경우) 및 기본 명령어

git config (컴퓨터에 깃허브 아이디,이메일,페스워드(토큰)을 등록하는 명령어)

git init (컴퓨터에 github repo를 연결하는 명령어)(git clone으로 할 경우 이 명령어를 사용할 필요 없음)

  • 깃에 연결할 폴더에 cd 명령어로 이동 후, 해당 명령어 실행 시 .git파일 생성(숨김 파일)됨
  • 동시에 깃명령어 수행하는 폴더로 변경(iterm2를 사용할 경우(커스터마이징) 브런치네임이 나옴)

git clone (보통 url.git)

  • git clone ../StudyforGit(repository name)
  • 상대방, 혹은 자신의 깃에 연결하여 다운을 받음
  • 폴더나 url에 연결 url에 보통 연결

git status

  • 파일 수정 후 상태 확인 창.(확인 할 필요는 없지만 초반에는 도움이 많이 되는 명령어)
  • 상대방이 수정 했을 경우에도 도움이 되니 초반에 많이 사용하도록 해보자.

git add ‘filename’ or . or ‘-A’

  • 파일 수정 후 페이지에 변경할 파일명 등록. ‘.’이나 ‘-A’입력 시 폴더 전체를 등록함

git commit -m ‘message’

git push

git pull

변경 로그 확인(log, diff)

add된 명령 취소하기(커밋 전)

git revert(push된 명령 취소하기)(로그 남음)

git reset(push된 명령 취소하기)

  • revert랑 유사하지만 reset은 로그를 남기지 않기때문에 revert가 보다 유용함

revert한것을 다시 되돌리고 싶을 경우
